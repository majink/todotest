# 업무 대장 · 워크리스트

스프레드시트(xlsx/csv)를 불러와 워크리스트로 자동 정리하고, 마감이 임박한 업무를 데스크탑 알림으로 알려주는 개인용 웹앱입니다. 서버나 로그인이 필요 없고, 데이터는 브라우저(로컬)에만 저장됩니다.

## 파일 구성 (최소화)

```
index.html      ← 화면 + 스타일 + 동작 로직 전부 포함 (사실상 이 파일 하나면 앱이 돕니다)
manifest.json    ← "데스크탑 앱처럼 설치"를 위한 PWA 설정 (별도 파일이어야 브라우저가 인식함, 앱 아이콘도 별도 이미지 파일 없이 이 안에 코드로 내장되어 있음)
sw.js            ← 오프라인 캐싱 + 설치 가능하게 해주는 서비스워커 (브라우저 규칙상 분리 필수)
template.xlsx    ← 담당자/마감일 형식이 맞춰진 기본 스프레드시트 템플릿
```
`manifest.json`과 `sw.js`는 웹 표준상 반드시 별도 파일이어야 브라우저가 "설치 가능한 앱"으로 인식하기 때문에 합치지 못했고, 그 외 화면·스타일·로직은 모두 `index.html` 한 파일에 합쳐놨습니다. 앱 아이콘은 별도 이미지 파일(png 등) 없이 `manifest.json` 안에 SVG 코드로 직접 넣어놨습니다 — 이미지 파일을 따로 준비/업로드할 필요가 없습니다.

## 배포 방법 — GitHub Pages (추천, 무료)

1. GitHub에 새 저장소 생성 (예: `worklist-app`)
2. 이 폴더의 파일 전체를 저장소에 업로드 (드래그 앤 드롭으로 가능)
3. 저장소 **Settings → Pages** 이동
4. **Source**를 `Deploy from a branch` → 브랜치 `main`, 폴더 `/ (root)` 선택 후 저장
5. 1~2분 후 `https://아이디.github.io/worklist-app/` 주소로 접속 가능
6. 이후 파일을 수정하고 싶으면 저장소에서 직접 수정하거나 다시 업로드하면 자동 반영됩니다

## 배포 방법 — Netlify (드래그 앤 드롭, 더 간단함)

1. [netlify.com](https://www.netlify.com) 무료 가입
2. 대시보드에서 **"Add new site" → "Deploy manually"**
3. 이 폴더를 통째로 마우스로 끌어다 놓기
4. 몇 초 후 `https://무작위이름.netlify.app` 같은 주소가 자동 생성됨 (Site settings에서 이름 변경 가능)

두 방법 다 무료 플랜으로 충분하고, 이후 계속 이 주소로 접속하면 됩니다. 로컬 서버를 매번 켤 필요가 없어집니다.

## 데스크탑 앱처럼 설치하기 (PWA)

1. 배포된 주소(또는 로컬 서버 주소)를 Chrome/Edge에서 접속
2. 주소창 오른쪽의 **설치 아이콘**(⊕ 모양) 클릭 → "설치"
   - 또는 Chrome 메뉴(⋮) → "앱 설치"
3. 설치하면 바탕화면/시작 메뉴에 아이콘이 생기고, 브라우저 주소창 없이 독립된 앱 창으로 실행됩니다

## Google Sheets 연동 (선택 사항 — 여러 기기·공유가 필요할 때)

파일 업로드 없이, 웹에서 입력하는 내용이 지정한 구글 시트에 바로 저장/동기화되게 할 수 있습니다. 아래는 한 번만 하면 되는 준비 과정입니다.

**1) 구글 시트 준비**
1. [sheets.google.com](https://sheets.google.com)에서 새 스프레드시트 생성
2. 첫 번째 탭 이름을 예: `업무목록`으로 변경
3. 1행에 헤더 입력: `제목 | 담당자 | 마감일 | 완료`
4. 주소창의 URL에서 `/d/`와 `/edit` 사이의 긴 문자열이 **스프레드시트 ID**입니다
   (`https://docs.google.com/spreadsheets/d/이_부분이_ID/edit`)

**2) Google Cloud Console에서 OAuth 클라이언트 ID 발급**
1. [console.cloud.google.com](https://console.cloud.google.com)에서 새 프로젝트 생성 (또는 기존 프로젝트 선택)
2. 왼쪽 메뉴에서 **"API 및 서비스" → "라이브러리"** → `Google Sheets API` 검색 후 **사용 설정**
3. **"API 및 서비스" → "OAuth 동의 화면"**에서 User Type을 `External`로 설정하고 앱 이름 등 최소 정보만 입력해 저장 (교직원 개인 사용이면 "테스트 사용자"에 본인 Google 계정 이메일을 추가)
4. **"API 및 서비스" → "사용자 인증 정보" → "사용자 인증 정보 만들기" → "OAuth 클라이언트 ID"**
   - 애플리케이션 유형: **웹 애플리케이션**
   - "승인된 자바스크립트 원본"에 실제 접속 주소 추가:
     - 로컬 테스트용: `http://localhost:8000`
     - 배포 후: `https://아이디.github.io` 또는 `https://사이트이름.netlify.app` (경로 없이 도메인만)
   - 생성된 **클라이언트 ID**(`xxxxx.apps.googleusercontent.com` 형식)를 복사

**3) 앱에서 연동 설정**
1. 화면 상단 **"⚙️ 연동 설정"** 클릭
2. 클라이언트 ID, 스프레드시트 ID, 탭 이름(`업무목록`)을 입력하고 **"저장하고 연결"**
3. 처음 한 번은 Google 로그인 창이 뜨고 "확인되지 않은 앱" 경고가 나올 수 있는데, 본인이 테스트 사용자로 등록했다면 "고급 → 이동"으로 계속 진행 가능
4. 연결되면 상단에 "Sheets 연결됨"이 표시되고, 이후 **"+ 항목 추가"**로 입력하는 내용이 바로 구글 시트에 저장됩니다. 시트를 직접 열어 수정해도 새로고침하면 반영됩니다.

**주의할 점**
- 이 모드에서는 스프레드시트 파일 업로드 기능 대신 구글 시트 자체가 데이터 저장소가 됩니다.
- 삭제는 시트에서 직접 행을 지우거나 앱의 ✕ 버튼으로 하면 되고, "전체 삭제" 버튼은 이 모드에서는 비활성화되어 있습니다 (구글 시트 데이터를 실수로 통째로 날리지 않도록).
- 다른 컴퓨터/교직원이 같은 데이터를 보려면, 같은 클라이언트 ID·스프레드시트 ID로 연동 설정하면 됩니다 (단, 그 사람도 시트에 대한 접근 권한이 있어야 함 — 시트 공유 설정에서 추가).
- "연동 설정 → 연결 해제"를 누르면 언제든 로컬 저장 모드로 되돌아갈 수 있습니다.



브라우저 보안 정책상 `index.html`을 더블클릭해서 열면 서비스워커·알림이 제한될 수 있어, 로컬 서버로 열어보는 걸 권장합니다.

```
python -m http.server 8000
```
또는
```
npx serve .
```
이후 `http://localhost:8000` 접속.

## 사용 방법

1. **스프레드시트 불러오기**로 xlsx/csv 업로드
   - 열 이름: `제목, 담당자, 마감일, 완료` (영문 title/owner/due/done도 인식)
   - **완료** 열은 텍스트 상태값이 아니라 체크 표시입니다: 칸에 아무 표시(`V` 등)를 하면 완료, 비워두면 미완료로 인식됩니다
   - 여러 시트가 있으면 왼쪽 "시트철" 탭으로 전환
2. **템플릿 내려받기**로 형식이 맞춰진 `template.xlsx`를 받아 그대로 채워 넣기 (완료 열은 드롭다운으로 V 선택 가능)
3. **알림 켜기**로 브라우저 알림 허용 → 마감 3일 이내이거나 기한을 넘긴 업무가 있으면 하루 한 번 알림
4. 표에서 동그라미 클릭 = 완료 처리, ✕ = 삭제
5. **+ 항목 추가**로 스프레드시트 없이 직접 입력도 가능

## 참고

- 데이터는 접속한 브라우저에만 저장됩니다(로컬 저장). 다른 컴퓨터에서 보려면 같은 스프레드시트를 다시 불러와야 합니다.
- 여러 사람이 같은 데이터를 실시간으로 같이 보려면 나중에 서버 + 클라우드 저장소를 추가하는 확장이 필요합니다 — 필요해지면 말씀해주세요.
